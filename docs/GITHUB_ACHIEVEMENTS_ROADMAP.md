# GitHub Achievements Roadmap

This roadmap tracks GitHub badges and achievements through real, useful work. The goal is to improve the profile, repositories, discussions, and open-source participation without spam or artificial activity.

## Immediate Achievement Targets

| Achievement | Legitimate action | Status |
|---|---|---|
| Quickdraw | Open and close a real maintenance issue within 5 minutes after the work is complete. | In progress |
| Pull Shark | Merge useful pull requests into a public repository. | In progress |
| YOLO | Merge an owned pull request without a review where that is appropriate. | In progress |

## Collaboration Targets

| Achievement | Legitimate action |
|---|---|
| Pair Extraordinaire | Co-author commits with another contributor on real changes that are merged through pull requests. |
| Galaxy Brain | Provide helpful accepted answers in GitHub Discussions. |
| Starstruck | Build and share projects that earn stars organically. |

## Profile Badge Targets

| Badge | Legitimate action |
|---|---|
| Developer Program Member | Join the GitHub Developer Program and build with GitHub APIs or Apps. |
| Campus Expert | Apply through GitHub Education when eligible. |
| Security Bug Bounty Hunter | Submit a valid GitHub Security Bug Bounty report. |
| Security advisory credit | Receive credit on a published GitHub security advisory for a real vulnerability. |
| Public Sponsor | Sponsor open source through GitHub Sponsors. This requires a manual paid action. |

## Historical Or Limited Badges

Some badges cannot be earned through normal current work. Examples include Arctic Code Vault Contributor and Mars 2020 Contributor.

## Ground Rules

- Use real issues, pull requests, discussions, and project improvements.
- Avoid spam, fake collaboration, or low-quality activity.
- Prefer visible improvements to profile quality, documentation, automation, tests, and open-source support.
- Treat external achievements as long-term participation goals rather than one-off tricks.
